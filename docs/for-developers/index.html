FAQ for developers

How does app blocking work technically?

AppBlocker uses NSWorkspace notifications to monitor app launches. When a blocked app is detected, it sends a SIGTERM to the process. For websites, it adds entries to /etc/hosts pointing the domain to 127.0.0.1.

Can I contribute?

Yes. It's MIT licensed on GitHub. PRs welcome. The codebase is intentionally small — if your feature can't be done in a few lines, it might not belong in the core. But fork it and go wild.

Does it need root access?

Website blocking requires sudo (to modify /etc/hosts). App blocking doesn't. The install script will explain what permissions are needed.

Will it conflict with my dev tools?

No. AppBlocker only blocks what you tell it to block. Your IDE, Terminal, browsers (except for blocked domains), and dev tools are untouched. Don't block Terminal though — that would be ironic.

Why Swift and not a shell script?

App blocking requires hooking into NSWorkspace for process monitoring, which needs a native macOS runtime. Swift gives us that with zero dependencies and compiles to a single binary. No Python, no Node, no Ruby runtime needed.
