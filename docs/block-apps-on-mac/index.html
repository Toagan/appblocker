  1. Install AppBlocker

    curl -fsSL https://raw.githubusercontent.com/Toagan/appblocker/main/install.sh | bash

  2. Block the app

    Use the app's name as it appears in your Applications folder:

    appblocker block "Slack"

  3. Try to open it

    Go ahead. Click the icon in your Dock. It opens for a split second and closes. AppBlocker is watching. The app is dead on arrival.

FAQ: Blocking apps on Mac

Can I block any Mac app?

Yes. Any app that runs as a process on your Mac can be blocked. This includes apps from the App Store, apps installed manually, and system apps (though we recommend not blocking Finder or Terminal).

What happens when I try to open a blocked app?

The app launches briefly and is immediately terminated by AppBlocker. You'll see it flash and close. It can't run while it's on the block list.

Is this better than macOS Screen Time?

For self-control purposes, yes. Screen Time lets you bypass limits with one click ("Ignore Limit"). AppBlocker requires you to open Terminal and run an unblock command — enough friction to make you think twice.

Can I block apps on a schedule?

Scheduled blocking is coming soon. For now, you can use cron jobs or macOS's built-in launchd to schedule block and unblock commands.

How do I unblock an app?

Run appblocker unblock "AppName" in Terminal. The app is immediately available again.

Does this collect any data?

No. Zero telemetry. AppBlocker runs entirely locally, makes no network requests, and is open source so you can verify this yourself. Learn more.
